Design Patterns

Isolation

Each agent has its own workspace, tools, memory scope, sessions, and LLM config

Config-Driven

Everything from providers to schedules to credentials lives in alfred.json

Provider Agnostic

Swap LLMs per-agent without touching agent code. Anthropic, xAI, OpenAI, Ollama

Zero-Config Auth

Domain-based credential auto-injection. Agents never see raw API keys

Memory-First

Vector memory with importance scoring (0-1), smart relevance gating (distance < 0.4), temporal decay, and outcome linking. Agents learn over time

Self-Learning

Post-session reflection loop evaluates performance. Confidence scores stored as importance. Decisions linked to outcomes for causal learning

Self-Modifying

Agents create new tools at runtime via meta-tools (tool_create, tool_remove)

Cost-Aware

Per-model USD cost estimates from token counts. Tool results auto-summarized when > 2K chars to save context. Daily cost alerts via Discord

Observable

Every interaction logged with tokens, latency, cost, tool calls. Vector queries tracked with result count and relevance scores

Resilient Execution

Auto-retry on transient errors (timeout, 429, 503) with exponential backoff. LLM fallback. Memory compaction. Metrics never block agents

Dashboard-Driven

Create agents, edit configs, and monitor metrics from the web UI. Optional LLM-generated SOUL.md

Proactive Reporting

Scheduled tasks automatically post results to their agent's Discord channel. No user prompt needed

Autonomous Trading

Standalone bot process with signal-based execution (BB+RSI), PID lifecycle management, historical seeding, CSV logging, and graceful shutdown with position exit

Process Management

Shared PID utilities (read, alive, kill-and-wait) in core/process.py. Used by daemon, Discord bot, and CLI for bot lifecycle
