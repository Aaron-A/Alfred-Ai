Design Patterns

Isolation

Each agent has its own workspace, tools, memory scope, sessions, and LLM config

Config-Driven

Everything from providers to schedules to credentials lives in alfred.json

Provider Agnostic

Swap LLMs per-agent without touching agent code. Anthropic, xAI, OpenAI, Ollama

Zero-Config Auth

Domain-based credential auto-injection. Agents never see raw API keys

Memory-First

Vector memory auto-recalled on every interaction. Agents learn over time

Self-Modifying

Agents create new tools at runtime via meta-tools (tool_create, tool_remove)

Observable

Every interaction logged with tokens, latency, tool calls to SQLite

Graceful Degradation

LLM fallback, scheduler exception isolation, metrics never block agents

Dashboard-Driven

Create agents, edit configs, and monitor metrics from the web UI. Optional LLM-generated SOUL.md

Proactive Reporting

Scheduled tasks automatically post results to their agent's Discord channel. No user prompt needed
